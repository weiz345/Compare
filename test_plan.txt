BruteForceNN equivalence test plan (indices only)
==================================================

Goal
----
Verify Python and C++ return the same top-k neighbor indices for the same
build/query sequences.

API under test
--------------
  build(points)
  query(point, k)  ->  list of Neighbor { index, distance }

We only compare the returned indices, in order. Distances are ignored.


Comparison rule
---------------
  For each query, the index list must match exactly.

  Example: query returns neighbors [(1, d1), (0, d2)]
           expected indices: [1, 0]


Script format (for later automation)
------------------------------------
  One shared script, run top-to-bottom. Each build replaces the index.

  build
  <point coords, one point per line>
  query <coords...> k <int>

  Runner output per query (indices only):
    query k 2
    1 0


Test 1: basic_knn
-----------------
  build: (0,0), (1,0), (0,1), (5,5)
  query (0.9, 0.1), k=2

  expected indices: [1, 0]


Test 2: empty_index
-------------------
  build: (empty)
  query (0, 0), k=3

  expected indices: []


Test 3: non_positive_k
----------------------
  build: (0,0), (1,1)
  query (0, 0), k=0
  query (0, 0), k=-1

  expected indices: [], []


Test 4: k_larger_than_n
-----------------------
  build: (0,0), (1,1)
  query (0, 0), k=10

  expected indices: [0, 1]


Test 5: k_equals_one
--------------------
  build: (0,0), (3,4), (5,5)
  query (0.1, 0.1), k=1

  expected indices: [0]


Test 6: query_exact_match
-------------------------
  build: (2,3), (7,8)
  query (2, 3), k=2

  expected indices: [0, 1]


Test 7: mismatched_dimensions
-----------------------------
  build: (0,0), (1,1)
  query (0, 0, 99), k=2

  expected indices: [0, 1]


Test 8: rebuild_replaces_index
------------------------------
  build: (0,0), (1,0)
  query (0.9, 0.1), k=1
  build: (5,5)
  query (0, 0), k=1

  expected indices: [1], [0]


Test 9: rebuild_to_empty
------------------------
  build: (0,0), (1,1)
  query (0, 0), k=1
  build: (empty)
  query (0, 0), k=1

  expected indices: [0], []


Test 10: single_point
---------------------
  build: (3,4)
  query (0, 0), k=1

  expected indices: [0]


Test 11: repeated_queries
-------------------------
  build: (0,0), (1,0), (0,1)
  query (0.9, 0.1), k=2
  query (0.1, 0.9), k=2

  expected indices: [1, 0], [2, 0]


Combined script order
---------------------
  Test 1 through 11, concatenated into one script.cases file.


Verification (later)
------------------
  1. Python runner executes script.cases, prints index lists.
  2. C++ runner executes script.cases, prints index lists.
  3. Diff the two outputs.


Out of scope
------------
  - distance values
  - tie ordering (tests chosen so top-k indices are unambiguous)
  - radius search, approximate methods, threading, errors
